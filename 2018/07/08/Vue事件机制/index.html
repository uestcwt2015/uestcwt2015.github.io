Vue事件机制

发表于 2018-07-08 • 分类于 JavaScript

源码分析Vue事件机制

vue 的事件都是通过v-on指令进行监听，这会涉及到vue是如何将template转化为render的，这里先不做讨论

在合成AST时，会对events进行处理

在Virtual DOM 渲染成真实DOM时会绑定事件

事件API

Vue为我们提供了四个api, 分别是
$on
$off
$once
$emit

初始化事件

vm.$on

用来监听当前实例上的自定义事件，该事件由$emit触发

vm.$off

用来移除自定义事件监听器

vm.$emit

触发当前实例的事件

vm.$once

监听一个自定义事件，但是只触发一次，在第一次触发之后移除监听器。

总结

  • 用v-on注册的事件都会经过处理成render函数
  • 注册的事件分为两大类，注册在普通HTML元素上的事件和注册在组件上的事件，但如果注册在组件上的事件有native修饰符，则会和普通HTML事件走相同的流程
  • 普通HTML事件和native的组件事件最终会通过addEventListener绑定在元素上
  • 组件事件则会通过$on去注册
